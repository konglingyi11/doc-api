# 验证配置教程

## 最快验证方法

按这个顺序做：

1. 保存配置。
2. 工具需要时，重启或重新加载。
3. 发送一个真实请求。
4. 查看返回内容或报错。

不要只看“保存成功”。保存成功不代表请求成功。

## 成功是什么样

成功时通常会看到：

- 工具正常返回内容。
- 没有报错。
- 连接测试显示成功。
- 日志里状态码是 200 或同类成功状态。

只要真实请求能返回内容，配置基本就是通的。

## 失败后看报错

### 401 Unauthorized

通常是 API Key 问题。

检查：

- Key 是否完整。
- Key 是否有多余空格或换行。
- Key 是否来自正确服务商。
- Key 是否过期、撤销或额度用完。

详见 [API Key 教程](/guide/api-key)。

### 404 Not Found

通常是 Base URL 或路径问题。

检查：

- 是否需要带 `/v1`。
- 是否多带了 `/v1`。
- 工具是否会自动补接口路径。
- 是否把完整路径填到了错误字段。

详见 [Base URL 教程](/guide/base-url)。

### Model Not Found

通常是模型名称问题。

检查：

- 模型名是否拼写正确。
- 大小写、连字符、日期版本是否正确。
- 你的 Key 是否有权限。
- 服务商是否还支持这个模型。

### Provider 或接口格式不匹配

通常是工具选择的接口和服务端接口不一致。

检查：

- 工具是否选了正确 Provider。
- 工具是否走 OpenAI Chat、OpenAI Responses 或 Anthropic Messages。
- Base URL 是否对应该接口格式。
- 模型是否适用于该接口。

## 进一步排查

如果还不通，继续查：

- 看工具日志里的请求地址和状态码。
- 使用工具自带的 Test Connection。
- 自建中转或本地代理时，看服务端日志。
- 换一个最简单的请求再试。

排查时一次只改一个字段。改完保存，再发真实请求。
