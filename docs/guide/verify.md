# 如何验证配置是否生效

最有效的验证方式是发一次真实请求。

## 最短验证流程

1. **保存配置**:确认配置文件已保存或设置面板已点"保存"
2. **重新加载**:有些工具需要重新加载配置或重启
3. **发真实请求**:发起一个简单对话或代码生成请求
4. **看结果**:成功返回内容就说明通了

## 成功时应该看到什么

- 工具正常返回响应内容
- 没有报错提示
- 有些工具会显示"连接成功"或绿色状态标识

## 失败时按报错类型分流

### 401 Unauthorized

说明 Key 有问题:

- Key 是否填对(完整、无多余空格、没有漏字符)
- Key 是否来自正确的服务商
- Key 是否还有效

→ 回到 [API Key 页](/guide/api-key) 排查

### 404 Not Found

说明地址有问题:

- baseURL 是否填对
- 是否需要带 `/v1` 路径(有些工具自动补,有些不补)
- 模型名是否填在 URL 里(某些工具会把模型名拼进路径)

→ 去对应工具页看 baseURL 的正确写法

### 模型错误 / Model Not Found

说明模型名有问题:

- 模型名是否拼写正确(大小写、连字符、版本号)
- 你的 Key 是否有权限访问这个模型
- 服务商是否还支持这个模型

→ 去服务商官网查当前可用的模型列表,或去工具页看示例

### Provider 错误 / 接口格式错误

说明接口格式不匹配:

- 工具期望的接口格式(OpenAI Chat / OpenAI Responses / Anthropic Messages)和你填的 baseURL 是否匹配
- 是否需要在工具里手动选择 Provider 类型

→ 去工具页看"接口格式"或"Provider"字段的说明

## 如果想进一步确认请求走到哪里

按优先级:

1. **看工具日志**:有些工具会打印请求地址和响应状态
2. **看连接测试**:如果工具有"Test Connection"按钮,点一下看提示
3. **看服务端日志**:如果你用的是自建中转或本地代理,去看那边的日志
4. **抓包**:高级用户可以用 Wireshark、Charles 等工具看实际请求地址

大多数情况下,发一次真实请求就能知道通了没有。报错类型会直接告诉你问题出在哪。
