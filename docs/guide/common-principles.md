# 通用配置原则

先按这个顺序判断，不要一上来就猜字段。

## 1. 先锁定你正在使用的工具

这是所有判断的起点。不同工具的字段名、接口格式、路径粒度都不同。你看到的某个教程里的"Base URL 填 xxx"只对那个工具有效，换一个工具可能就错了。

打开工具页，确认版本号，看清楚它提供的字段示例。不要凭借在别处看到的配置经验直接往里填。

## 2. 再判断这个工具接哪种接口格式

接口格式决定了很多东西：字段怎么叫、路径怎么拼、模型名怎么填。主流格式有三类：

- **OpenAI Chat**：最常见，很多工具都兼容。Base URL 通常需要 `/v1/chat/completions` 路径，但具体要不要填 `/v1` 取决于工具。
- **OpenAI Responses**：新格式，路径通常是 `/v1/responses`。
- **Anthropic Messages**：Anthropic 原生格式，路径是 `/v1/messages`。

工具页通常会写清楚它走哪种接口。如果没写，先在工具页里找线索，不要靠猜。

## 3. 不要把 baseURL 当固定答案去背

baseURL 没有一个全站通用固定答案。它填到什么粒度（域名、域名带 `/v1`、还是完整路径），取决于：

- 工具帮你补了哪些路径
- 工具期望你填到哪一级

有些工具你填 `https://api.example.com` 它自己会加 `/v1/chat/completions`；有些工具你填到 `/v1` 它才对；有些工具要求你填完整路径。

所以：先看当前工具页的示例，不要把别处学来的 baseURL 直接复制过来。

## 4. 字段名以工具页和当前版本界面为准

不同工具对相同概念可能叫不同名字：

- API Key 可能叫 `apiKey`、`api_key`、`API Key`、`Key`
- Base URL 可能叫 `baseURL`、`base_url`、`Base URL`、`API Endpoint`

而且同一个工具不同版本可能改名。所以：打开你正在用的工具版本，看它界面上的字段名，不要凭记忆填。

## 5. 模型名只在需要时填

模型名不是必填项。很多工具会自动拉取可用模型列表，你直接选就行。

如果需要手动填：

- 优先选最新的可用模型（工具页通常会列推荐模型）
- 不确定有哪些模型时，去服务商的模型广场确认
- 不要填已经下线的模型

## 6. 配完以后一定做一次真实验证

填完不等于配对。验证方法：

- 发一个真实请求，看是否成功返回
- 用工具自带的"测试连接"按钮
- 查工具日志看实际发出的请求路径

验证方法详见 [如何验证配置是否生效](/guide/verify)。

## 7. 遇到报错先看报错类型，不要盲改

不同报错指向不同问题：

- **401 Unauthorized**：API Key 错了，回去查 Key
- **404 Not Found**：baseURL 或路径粒度错了，回头查 `/v1` 和路径拼接
- **模型不存在 / model not found**：模型名错了，查可用模型列表
- **provider 错误 / 不支持的接口**：接口格式选错了，查工具页确认它走哪种格式

不要看到一个报错就三个字段轮着改。先看报错类型，定位到具体字段，再改那一个。
