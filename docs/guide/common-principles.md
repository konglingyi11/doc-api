# 通用配置原则

## 先确认接口格式

先看工具接哪种接口。

常见接口格式：

- OpenAI Chat：使用 `messages`。
- OpenAI Responses：使用 `input`。
- Anthropic Messages：使用 Claude 风格的 `messages` 和请求头。
- image2：使用图片生成或图片编辑接口，不是 chat messages。

接口格式错了，字段填对也可能请求失败。

## 再确认 Base URL

Base URL 是 API 地址。

先看工具页示例，再决定要不要带 `/v1`。

有些工具只要域名。

有些工具要填到 `/v1`。

有些工具要填完整路径。

更多说明见 [Base URL 教程](/guide/base-url)。

## 然后填写 API Key

API Key 要填到密钥字段。

不要多空格。

不要换行。

不要把其他服务商的 Key 填进来。

更多说明见 [API Key 教程](/guide/api-key)。

## 最后填写模型名称

模型名称要和服务商支持的名称一致。

注意大小写、连字符和日期版本。

如果工具可以拉取模型列表，优先从列表里选。

如果需要手动填，按工具页或服务商页面的写法填。

## 不要混用配置

不要把一个工具的配置直接搬到另一个工具。

重点检查：

- Provider 是否选对。
- Base URL 是否按当前工具写。
- API Key 是否来自同一个服务商。
- 模型名称是否适用于当前接口。

## 配好后立即测试

保存配置后，发一次真实请求。

如果工具需要重启或重新加载，先重启或重新加载。

成功返回内容，才算配置生效。

失败时按报错查。详见 [验证配置教程](/guide/verify)。
