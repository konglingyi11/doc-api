# 常见问题

这里只回答跨工具都会遇到的问题。具体字段怎么填,还是回到对应工具页看最快。

## 我应该先看哪一页?

按这个顺序:

1. **先看目标工具页**:你要配置哪个工具,就直接去那个工具的页面
2. **遇到 Key 问题**:回来查 [API Key 页](/guide/api-key)
3. **验证是否生效**:回来查 [验证页](/guide/verify)
4. **遇到跨工具问题**:看本页下面的常见问题

## 为什么有些工具接 OpenAI Chat,有些接 OpenAI Responses,有些接 Anthropic Messages?

因为不同工具适配的接口格式不同:

- **OpenAI Chat**:最通用的格式,大部分工具都支持
- **OpenAI Responses**:某些工具(如 OpenAI 官方工具)使用的新格式
- **Anthropic Messages**:Anthropic 原生接口格式,某些工具原生支持

工具页会明确写它支持哪种格式。你需要在工具里选择正确的 Provider 或接口类型,然后按工具页的示例填 baseURL 和其他字段。

## 为什么有的工具 baseURL 要带 /v1,有的不用?

取决于工具的实现:

- 有些工具会自动在 baseURL 后面补 `/v1/chat/completions` 或 `/v1/messages`
- 有些工具不会补,需要你自己在 baseURL 里带完整路径

工具页会写清楚 baseURL 应该填成什么样。不要自己猜,直接按示例填。

## 为什么我填了 Key 还是报 401?

最常见的 4 个原因:

1. **Key 复制错了**:多了空格、少了字符、复制了换行符
2. **Key 来源不匹配**:OpenAI 的 Key 填到了 Anthropic 字段,或反过来
3. **Key 已失效**:被撤销、过期、或用完了额度
4. **字段名填错**:某些工具区分 "API Key" 和 "API Secret"

→ 去服务商控制台确认 Key 状态,然后对照工具页示例重新填

## 为什么我改了地址却报 404?

最常见的 3 个原因:

1. **baseURL 拼错了**:多了或少了路径,特别是 `/v1` 这一段
2. **模型名拼在 URL 里了**:某些工具会把模型名拼进路径,导致地址错误
3. **协议不对**:用了 `http://` 但接口要求 `https://`,或反过来

→ 对照工具页的 baseURL 示例,逐字符检查

## 为什么模型名填了还是不生效?

常见原因:

1. **模型名拼写错误**:大小写、连字符、版本号都要完全匹配
2. **模型不可用**:你的 Key 没有权限访问这个模型,或服务商已下架
3. **字段名错误**:填错了位置(比如填成了 "Model ID" 而不是 "Model Name")

→ 去服务商官网查当前可用模型列表,或直接用工具页示例里的模型名

## 为什么改完配置没有马上生效?

常见原因:

1. **没保存**:配置文件改了但没保存,或设置面板没点"保存"
2. **没重新加载**:某些工具需要手动重新加载配置或重启
3. **改错文件**:改了全局配置但工具读的是项目配置,或反过来

→ 确认保存后,尝试重新加载或重启工具

## 我应该优先看最新教程还是旧帖子?

永远优先看:

1. **工具官方文档**:最权威,更新最快
2. **本站的工具配置页**:我们按最新版本更新
3. **服务商官方文档**:API 变更、模型下架等信息会第一时间公告

旧帖子(包括论坛、博客、问答网站)可能过时了。接口地址、模型名、字段名都可能已经变化。如果旧帖子和官方文档冲突,以官方文档为准。
